# 기여 가이드

## 브랜치 & PR
- `main` 직접 push 금지. **브랜치 → PR → 리뷰 → 머지.**
- PR 은 이슈와 연결한다 (`Closes #이슈번호`).

## 커밋 메시지 컨벤션

Conventional Commits 기반. **제목 끝에 이슈번호를 붙인다.**

```
<type>: <제목> (#이슈번호)
```

예시:
```
feat: 로그인 화면 추가 (#42)
fix: 캔버스 드래그 끊김 수정 (#51)
refactor: 그룹 리스트 Store 상태 평탄화 (#63)
```

| type | 의미 | type | 의미 |
|------|------|------|------|
| `feat` | 기능 추가, 버그 수정 |
| `refactor` | 동작 변화 없는 개선, 성능개선 | 
| `docs` | 문서 |

규칙:
- 제목은 50자 내외, 명령형(추가/수정).
- 제목 끝 `(#번호)` 는 **필수** — PR·이슈에 자동 링크된다.
- 본문(선택)은 *무엇을·왜* 바꿨는지. *어떻게*는 코드가 설명한다.

## 커밋 템플릿 설정 (1회)

저장소에 포함된 `.gitmessage` 를 커밋 템플릿으로 등록하면, 커밋할 때 양식이 에디터에 자동 표시된다.

```sh
git config commit.template .gitmessage
```
